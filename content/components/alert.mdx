## Variants

Alert supports four semantic variants that determine color, icon, and ARIA role.

### Info (default)

Used for informational messages, tips, and notes. Renders with `role="status"` (polite live region).

<ComponentPage
	title="Info Variant"
	description="Default variant for informational messages, tips, and notes."
	preview={
		<div className="w-full">
			<Alert variant="info" title="Did you know?">
				You can drag and drop files to upload them.
			</Alert>
		</div>
	}
	code={`<Alert variant="info" title="Did you know?">
  You can drag and drop files to upload them.
</Alert>`}
/>

## Dismissible

Add the `dismissible` prop to show a close button. Provide an `onDismiss` callback to control visibility externally, or let the alert manage its own visibility.

<ComponentPage
	title="Dismissible Alerts"
	description="Alerts with a close button. Can be self-managed or controlled externally."
	preview={
		<div className="flex flex-col gap-3 w-full">
			<Alert variant="danger" dismissible>
				Something went wrong. Please try again.
			</Alert>
			<Alert variant="info" dismissible>
				This alert will hide itself when dismissed.
			</Alert>
		</div>
	}
	code={`// Controlled
<Alert variant="danger" dismissible onDismiss={() => setVisible(false)}>
  Something went wrong. Please try again.
</Alert>

// Self-managed (hides itself when dismissed)

<Alert variant="info" dismissible>
  This alert will hide itself when dismissed.
</Alert>`}
/>

## Props

| Prop           | Type                                           | Default            | Description                                                        |
| -------------- | ---------------------------------------------- | ------------------ | ------------------------------------------------------------------ |
| `variant`      | `"info" \| "success" \| "warning" \| "danger"` | `"info"`           | Semantic variant determining color, icon, and ARIA role.           |
| `title`        | `ReactNode`                                    | —                  | Title text displayed prominently at the top.                       |
| `icon`         | `ReactNode \| null`                            | Auto (per variant) | Custom icon. Pass `null` to hide.                                  |
| `dismissible`  | `boolean`                                      | `false`            | Whether to show a close button.                                    |
| `onDismiss`    | `() => void`                                   | —                  | Callback when dismissed. If omitted, alert self-manages.           |
| `dismissLabel` | `string`                                       | `"Dismiss alert"`  | Accessible label for the dismiss button.                           |
| `role`         | `"alert" \| "status"`                          | Auto (per variant) | ARIA role. Danger/warning default to `"alert"`, others `"status"`. |
| `className`    | `string`                                       | —                  | Additional CSS classes merged on the root element.                 |
| `children`     | `ReactNode`                                    | —                  | Description / body content.                                        |

## Accessibility

- **Danger** and **warning** alerts use `role="alert"` (assertive live region) — screen readers announce them immediately.
- **Info** and **success** alerts use `role="status"` (polite live region) — announced at the next opportunity.
- Each variant renders a unique icon so meaning is **never conveyed by color alone** (WCAG 1.4.1).
- The dismiss button has a configurable `aria-label` (defaults to `"Dismiss alert"`).
- Icons are decorative and marked with `aria-hidden="true"`.
