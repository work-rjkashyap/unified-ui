## Overview

The `Tooltip` component provides contextual information when users hover over or focus on an element. It's built on Radix UI's Tooltip primitive with design system tokens for consistent styling.

---

## Import

```tsx
import { Tooltip, TooltipProvider } from "@work-rjkashyap/unified-ui";
// or
import { Tooltip, TooltipProvider } from "@work-rjkashyap/unified-ui/components";
```

---

## Setup

Wrap your application (or a section of it) with `TooltipProvider` to enable tooltips. This configures shared delay and skip behavior for all tooltips within its scope.

```tsx
import { TooltipProvider } from "@work-rjkashyap/unified-ui";

export default function RootLayout({ children }) {
  return (
    <TooltipProvider>
      {children}
    </TooltipProvider>
  );
}
```

## Basic Usage

Wrap any element with `Tooltip` and provide a `content` prop. The tooltip appears on hover and focus.

```tsx
import { Tooltip, Button } from "@work-rjkashyap/unified-ui";

<Tooltip content="Save your changes">
  <Button>Save</Button>
</Tooltip>
```

<Callout type="info">
  The trigger element must be a single React element that can accept a ref. If you're wrapping a custom component, make sure it forwards refs.
</Callout>

## Placement

Control which side of the trigger the tooltip appears on with the `side` prop.

```tsx
<Tooltip content="Top tooltip" side="top">
  <Button variant="secondary">Top</Button>
</Tooltip>

<Tooltip content="Right tooltip" side="right">
  <Button variant="secondary">Right</Button>
</Tooltip>

<Tooltip content="Bottom tooltip" side="bottom">
  <Button variant="secondary">Bottom</Button>
</Tooltip>

<Tooltip content="Left tooltip" side="left">
  <Button variant="secondary">Left</Button>
</Tooltip>
```

| Side | Description |
|------|-------------|
| `top` | Above the trigger (default) |
| `right` | To the right of the trigger |
| `bottom` | Below the trigger |
| `left` | To the left of the trigger |

## Alignment

Fine-tune the tooltip position with the `align` prop.

```tsx
<Tooltip content="Start aligned" side="bottom" align="start">
  <Button variant="secondary">Start</Button>
</Tooltip>

<Tooltip content="Center aligned" side="bottom" align="center">
  <Button variant="secondary">Center</Button>
</Tooltip>

<Tooltip content="End aligned" side="bottom" align="end">
  <Button variant="secondary">End</Button>
</Tooltip>
```

## Arrow

By default, tooltips show a small arrow pointing toward the trigger. Set `arrow={false}` to hide it.

```tsx
{/* With arrow (default) */}
<Tooltip content="With arrow" arrow>
  <Button>Hover me</Button>
</Tooltip>

{/* Without arrow */}
<Tooltip content="No arrow" arrow={false}>
  <Button>Hover me</Button>
</Tooltip>
```

## Delay

Configure the open delay to control how quickly the tooltip appears. The `TooltipProvider` controls the shared delay for all tooltips in its scope.

```tsx
{/* Fast â€” 200ms delay */}
<TooltipProvider delayDuration={200}>
  <Tooltip content="Quick tooltip">
    <Button>Fast</Button>
  </Tooltip>
</TooltipProvider>

{/* Slow â€” 700ms delay */}
<TooltipProvider delayDuration={700}>
  <Tooltip content="Slow tooltip">
    <Button>Slow</Button>
  </Tooltip>
</TooltipProvider>
```

You can also override the delay per tooltip:

```tsx
<Tooltip content="Custom delay" delayDuration={0}>
  <Button>Instant</Button>
</Tooltip>
```

## Rich Content

The `content` prop accepts any `ReactNode`, so you can render rich tooltip content.

```tsx
<Tooltip
  content={
    <div className="flex flex-col gap-1">
      <p className="font-semibold text-xs">Keyboard Shortcut</p>
      <p className="text-xs text-ds-muted-foreground">
        Press <kbd className="px-1 py-0.5 bg-ds-muted rounded text-[10px]">Ctrl+S</kbd> to save
      </p>
    </div>
  }
>
  <Button>Save</Button>
</Tooltip>
```

## Max Width

Tooltips have a default max width of 220px to keep content readable. Override with `maxWidth`.

```tsx
<Tooltip
  content="This is a very long tooltip message that would normally wrap at the default max width."
  maxWidth={300}
>
  <Button>Wide tooltip</Button>
</Tooltip>
```

## With Icon-Only Buttons

Tooltips are especially useful with icon-only buttons to provide accessible labels.

```tsx
import { Settings, Trash, Copy, Download } from "lucide-react";

<Tooltip content="Settings">
  <Button variant="ghost" iconOnly aria-label="Settings">
    <Settings className="size-4" />
  </Button>
</Tooltip>

<Tooltip content="Delete">
  <Button variant="ghost" iconOnly aria-label="Delete">
    <Trash className="size-4" />
  </Button>
</Tooltip>

<Tooltip content="Copy to clipboard">
  <Button variant="ghost" iconOnly aria-label="Copy">
    <Copy className="size-4" />
  </Button>
</Tooltip>

<Tooltip content="Download">
  <Button variant="ghost" iconOnly aria-label="Download">
    <Download className="size-4" />
  </Button>
</Tooltip>
```

## Toolbar Example

A common pattern combining tooltips with a row of icon-only buttons.

```tsx
<div className="flex items-center gap-1 p-1 border border-ds-border rounded-ds-md">
  <Tooltip content="Bold (Ctrl+B)">
    <Button variant="ghost" size="sm" iconOnly aria-label="Bold">
      <BoldIcon className="size-4" />
    </Button>
  </Tooltip>
  <Tooltip content="Italic (Ctrl+I)">
    <Button variant="ghost" size="sm" iconOnly aria-label="Italic">
      <ItalicIcon className="size-4" />
    </Button>
  </Tooltip>
  <Tooltip content="Underline (Ctrl+U)">
    <Button variant="ghost" size="sm" iconOnly aria-label="Underline">
      <UnderlineIcon className="size-4" />
    </Button>
  </Tooltip>
</div>
```

## Props

### Tooltip

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `content` | `ReactNode` | **required** | The tooltip content. Can be a string or React node. |
| `side` | `"top" \| "right" \| "bottom" \| "left"` | `"top"` | Which side the tooltip appears on. |
| `align` | `"start" \| "center" \| "end"` | `"center"` | Alignment along the side axis. |
| `arrow` | `boolean` | `true` | Whether to show the arrow indicator. |
| `sideOffset` | `number` | `4` | Distance in px from the trigger. |
| `maxWidth` | `number` | `220` | Maximum width of the tooltip in px. |
| `delayDuration` | `number` | â€” | Override the provider's delay for this tooltip. |
| `children` | `ReactElement` | **required** | The trigger element (must accept a ref). |

### TooltipProvider

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `delayDuration` | `number` | `400` | Open delay in ms for all tooltips in scope. |
| `skipDelayDuration` | `number` | `300` | Time in ms before the delay resets after leaving a tooltip. |
| `disableHoverableContent` | `boolean` | `false` | Prevent tooltip from staying open when hovering over its content. |

## Accessibility

<Callout type="info">
  Built on **@radix-ui/react-tooltip** which provides full keyboard and screen reader support out of the box.
</Callout>

- **Keyboard accessible** â€” Tooltips appear when the trigger receives keyboard focus and hide on blur.
- **Escape to dismiss** â€” Pressing `Escape` closes the tooltip immediately.
- **ARIA attributes** â€” The tooltip is linked to its trigger via `aria-describedby` (managed by Radix).
- **Screen reader support** â€” The tooltip content is announced when the trigger is focused.
- **No focus steal** â€” Tooltips don't trap focus. They supplement existing content without blocking interaction.
- **Hover intent** â€” Configurable delay prevents tooltips from flickering during normal mouse movement.
- **Skip delay** â€” After viewing one tooltip, subsequent tooltips open faster (controlled by `skipDelayDuration`).
- The tooltip uses `z-ds-tooltip` for proper layering above page content.
- Animations are CSS-based (fade + scale) and respect `prefers-reduced-motion`.
