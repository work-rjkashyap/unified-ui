## Overview

Unified UI uses a two-tier color system:

1. **Base palettes** — Raw color ramps from 50 → 950 (inspired by Tailwind's palette structure).
2. **Semantic mappings** — Intent-based aliases (`primary`, `danger`, `success`, etc.) that map to specific palette stops for light and dark modes.

All color values are stored as **RGB channel strings** (e.g. `"15 23 42"`) so they work with Tailwind's opacity modifier syntax: `bg-ds-primary/50`.

<Callout type="info">
  Components never reference base palette values directly. They use semantic tokens through CSS custom properties (`--ds-color-primary`, `--ds-color-danger`, etc.), which the theme layer resolves to the correct palette stop for the active mode.
</Callout>

```tsx
import { brand, blue, green, amber, red, teal, neutral, palettes } from "@work-rjkashyap/unified-ui/tokens";

// Individual palette
console.log(brand[600]); // "79 70 229"

// All palettes bundled (useful for tooling / documentation)
console.log(Object.keys(palettes)); // ["slate", "gray", "zinc", "brand", "blue", "green", "amber", "red", "teal"]
```

## Contrast Compliance

All semantic color pairings are designed to meet **WCAG AA** contrast requirements:

- **Normal text** (< 18px or < 14px bold): 4.5:1 minimum contrast ratio
- **Large text** (≥ 18px or ≥ 14px bold): 3:1 minimum contrast ratio
- **Non-text elements** (borders, icons, form controls): 3:1 minimum per SC 1.4.11

### Key Decisions

- **Light mode borders** use custom RGB values (`rgb(188 188 194)` for `border`, `rgb(148 148 157)` for `input`) instead of palette stops to ensure they meet the 3:1 non-text contrast requirement against white backgrounds.
- **Dark mode primary** uses `brand.400` instead of `brand.600` to achieve 6.67:1 contrast against the dark background.
- **Disabled states** exceed WCAG requirements (which exempt disabled controls) with a ≥ 3:1 target for usability.

### Built-in Contrast Checking

Unified UI ships a contrast-checking utility you can use in tests or at build time:

```tsx
import { checkHexContrast, meetsAA, contrastRatio } from "@work-rjkashyap/unified-ui/utils";

// Check a specific pair
const result = checkHexContrast("#4F46E5", "#FFFFFF");
console.log(result.ratio);    // 6.35
console.log(result.aa);       // true (normal text)
console.log(result.aaLarge);  // true (large text)

// Quick boolean check
console.log(meetsAA(6.35, "normal")); // true

// Raw ratio calculation
console.log(contrastRatio("#000000", "#FFFFFF")); // 21
```

### Auditing All Semantic Pairs

```tsx
import { auditContrast, DS_LIGHT_CRITICAL_PAIRS, DS_DARK_CRITICAL_PAIRS } from "@work-rjkashyap/unified-ui/utils";

// Audit all critical light mode pairs
const lightResults = auditContrast(DS_LIGHT_CRITICAL_PAIRS);
const failures = lightResults.filter(r => !r.passes);

if (failures.length > 0) {
  console.error("Contrast failures:", failures);
}
```

## Usage Patterns

### Backgrounds & Text

```tsx
// Page background with default text
<div className="bg-ds-background text-ds-foreground">
  <h1>Page Title</h1>
  <p className="text-ds-muted-foreground">Secondary description text</p>
</div>

// Card surface
<div className="bg-ds-surface-raised border border-ds-border rounded-ds-md p-ds-4">
  Card content
</div>
```

### Status Colors

```tsx
// Success alert
<div className="bg-ds-success-muted text-ds-success-muted-foreground">
  Operation completed successfully.
</div>

// Danger button
<button className="bg-ds-danger text-ds-danger-foreground hover:bg-ds-danger/90">
  Delete
</button>

// Warning badge
<span className="bg-ds-warning-muted text-ds-warning-muted-foreground">
  Caution
</span>
```

### Opacity Modifier

Because color values are stored as RGB channels, you can use Tailwind's opacity modifier syntax:

```tsx
// 50% opacity primary background
<div className="bg-ds-primary/50" />

// 20% opacity border
<div className="border border-ds-border/20" />

// 80% opacity text
<p className="text-ds-foreground/80">Slightly transparent text</p>
```

### CSS Custom Properties

All semantic colors are available as CSS variables for use in custom styles:

```css
.custom-element {
  background-color: rgb(var(--ds-color-primary));
  color: rgb(var(--ds-color-primary-foreground));
  border-color: rgb(var(--ds-color-border));
}

/* With opacity */
.custom-overlay {
  background-color: rgb(var(--ds-color-primary) / 0.1);
}
```

## Importing Colors

```tsx
// Semantic mappings (what components use)
import { semanticLight, semanticDark } from "@work-rjkashyap/unified-ui/tokens";

console.log(semanticLight.primary);          // "79 70 229" (brand.600)
console.log(semanticDark.primary);           // "129 140 248" (brand.400)

// Raw palettes (for tooling, docs, custom themes)
import { brand, blue, green, amber, red, neutral } from "@work-rjkashyap/unified-ui/tokens";

console.log(brand[600]);   // "79 70 229"
console.log(red[500]);     // "239 68 68"
console.log(neutral[100]); // "244 244 245"
```

<Callout type="warn">
  **Don't use raw palette values in components.** Always use the semantic tokens (`bg-ds-primary`, not `bg-[rgb(79_70_229)]`). Semantic tokens automatically adapt to light/dark mode and can be rebranded without touching component code.
</Callout>
